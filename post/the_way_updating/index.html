更新时需要用到的命令

Feb 16, 2023
1 minute read

新手上路，难免不熟练，每次更新都要去搜一搜这些命令，所以这里做一个简单的汇总，以后直接看这篇就好了。

首先打开git bash，跳转至blog文件夹，注意这里是/而不是\。

$ cd F:/GitHub/Hugo/Sites/blog

通过 hugo new 命令新建名为 .md 的 markdown 文件。

$ hugo new post/.md

打开相应文件开始劈里啪啦一顿写。

写完了之后回到git bash，生成静态网页，静态网页的位置在blog\public。

$ hugo --baseUrl="https://auold.github.io/"

之后进入public文件夹进行推送。

$ cd public

这个文件夹本身已经初始化过了，经过试验可以不需要重新git init。先将所有文件添加到索引。

$ git add -A

提交代码，““中的内容表示注释说明，必须要有，用于注释这次提交是做了什么修改。

$ git commit -m ""

将代码推送至远程仓库。

$ git push

最后静静地等待github的workflow运行起来，再把先前的workflow删除就好。（我其实并不清楚不删除会怎样，目前看起来似乎没有问题，不过总感觉删了总比不删来得清楚）

希望能维持住更新。
